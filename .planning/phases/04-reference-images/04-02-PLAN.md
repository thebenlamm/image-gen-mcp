**6. Add reference-aware provider selection AFTER the existing size-aware selection:**

The current logic is:
1. Pick provider (explicit or default)
2. Size-aware selection (if size specified and provider doesn't support it)

Add a third step:
3. Reference-aware selection (if referenceBuffer provided and provider doesn't support it)

```typescript
// Reference-aware selection: if reference provided and provider doesn't support it,
// find a reference-capable provider respecting configured order
if (imageProvider && referenceBuffer && !imageProvider.supportsReference) {
  const referenceCapable = registry.getReferenceCapable();
  // Use configured order, falling back to any available reference-capable provider
  const orderedProvider = REFERENCE_PROVIDER_ORDER.find(
    name => referenceCapable.includes(name) && registry.has(name)
  ) || referenceCapable[0];

  if (orderedProvider) {
    providerName = orderedProvider;
    imageProvider = registry.get(providerName);
  }
}
```

Place this AFTER the existing size-aware selection block but BEFORE the `if (!imageProvider)` check.

**IMPORTANT:** When BOTH size AND reference are needed, reference takes priority. The reference-aware selection runs second, potentially overriding the size-aware selection. This is correct because reference-capable providers (OpenAI, Gemini) also support size.
