**3. Add the `generateWithReference()` private method:**

```typescript
private async generateWithReference(
  params: GenerateParams,
  model: string,
  size: string
): Promise<GenerateResult> {
  // Convert Buffer to a File-like object for the OpenAI SDK
  const imageFile = new File(
    [params.referenceImage!],
    'reference.png',
    { type: 'image/png' }
  );

  // Build prompt with reference weight hint
  const weight = params.referenceWeight ?? 0.8;
  let editPrompt = params.prompt;
  if (weight >= 0.7) {
    editPrompt = `Generate a new image closely matching the style and appearance of the reference image. ${params.prompt}`;
  } else if (weight >= 0.4) {
    editPrompt = `Generate a new image inspired by the reference image. ${params.prompt}`;
  } else {
    editPrompt = `Generate a new image loosely based on the reference image. ${params.prompt}`;
  }

  // Note: mask is intentionally omitted. The OpenAI images.edit API does NOT require
  // the mask parameter for gpt-image-1. Omitting mask makes the model edit the entire
  // image based on the prompt, which is the desired behavior for reference-based generation.
  const response = await this.client.images.edit({
    model,
    image: imageFile,
    prompt: editPrompt,
    n: 1,
    size: size as '256x256' | '512x512' | '1024x1024' | '1536x1024' | '1024x1536' | 'auto',
  });

  const imageData = response.data?.[0];
  if (!imageData?.b64_json) {
    throw new Error('No image data returned from OpenAI edit API');
  }

  const buffer = Buffer.from(imageData.b64_json, 'base64');

  return {
    buffer,
    model,
    revisedPrompt: imageData.revised_prompt,
  };
}
```

**Key decisions:**
- `referenceWeight` is encoded as a prompt hint (not an API parameter) because OpenAI's edit API doesn't have a direct weight parameter. The prompt phrasing guides how closely the output matches the reference.
- The `images.edit` endpoint returns b64_json for gpt-image-1, same as generate.
- The reference image Buffer is wrapped in a `File` object which satisfies the `Uploadable` type in the OpenAI SDK.
- Size mapping uses the edit API's supported sizes (which includes 'auto' and the same standard sizes).

**IMPORTANT:** The existing generate path must remain completely unchanged. The only difference is the conditional branch at the top of `generate()`.

```typescript
async generate(params: GenerateParams): Promise<GenerateResult> {
  const model = params.model || this.defaultModel;
  const aspectRatio = ASPECT_RATIO_MAP[params.size || 'square'];

  const imagenModel = this.client.getGenerativeModel({
    model,
  });

  // Build content parts
  const parts: Array<{ text: string } | { inlineData: { mimeType: string; data: string } }> = [];

  // Add reference image as first part if provided
  if (params.referenceImage) {
    const weight = params.referenceWeight ?? 0.8;
    let referenceInstruction: string;
    if (weight >= 0.7) {
      referenceInstruction = 'Generate a new image that closely matches the character, style, and appearance shown in this reference image.';
    } else if (weight >= 0.4) {
      referenceInstruction = 'Generate a new image inspired by the character and style shown in this reference image.';
    } else {
      referenceInstruction = 'Generate a new image loosely based on the reference image below.';
    }

    parts.push({
      inlineData: {
        mimeType: 'image/png',
        data: params.referenceImage.toString('base64'),
      },
    });
    parts.push({ text: `${referenceInstruction} ${params.prompt}` });
  } else {
    parts.push({ text: params.prompt });
  }

  const response = await imagenModel.generateContent({
    contents: [{ role: 'user', parts: parts as any }],
    generationConfig: {
      // @ts-expect-error - Imagen-specific config
      responseModalities: params.referenceImage ? ['TEXT', 'IMAGE'] : ['IMAGE'],
      aspectRatio,
    },
  });

  const result = response.response;
  const candidate = result.candidates?.[0];

  if (!candidate?.content?.parts?.[0]) {
    throw new Error('No image data returned from Gemini');
  }

  // When using multi-modal output, image may not be the first part
  // Find the first part with inlineData
  let imagePart = null;
  for (const part of candidate.content.parts) {
    if ('inlineData' in part && part.inlineData?.data) {
      imagePart = part;
      break;
    }
  }

  if (!imagePart || !('inlineData' in imagePart) || !imagePart.inlineData?.data) {
    throw new Error('No image data in Gemini response');
  }

  const buffer = Buffer.from(imagePart.inlineData.data, 'base64');

  return {
    buffer,
    model,
  };
}
```

**Key decisions:**
- Reference image is passed as `inlineData` with `mimeType: 'image/png'` and base64-encoded data. This matches the Gemini API's multi-modal content structure.
- When reference is provided, `responseModalities` includes `['TEXT', 'IMAGE']` because Gemini may return text alongside the image in multi-modal mode.
- The response parsing now searches ALL parts for image data (not just `parts[0]`) because in multi-modal mode, Gemini may return text first then image.
- `referenceWeight` is encoded as a prompt instruction (same strategy as OpenAI) since Gemini doesn't have a native weight parameter.
- The reference image part comes BEFORE the text prompt to give it priority in the model's attention.

**IMPORTANT:** When no referenceImage is provided, the behavior must be identical to the current implementation. The only change in the non-reference path is that response parsing now iterates parts (more robust, still finds the image in parts[0] as before).
