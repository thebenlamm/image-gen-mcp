Primary risks:
- Local input image paths can expose arbitrary files if the caller passes sensitive paths. For this personal MCP tool, Phase 5 accepts explicit local paths by design, but errors must identify missing/unreadable files without echoing file contents.
- OpenAI edit calls can fail after reading the input. The capability must surface provider errors without writing partial output.
- Capability functions must return buffers only; saving paths belongs in `image_op` so run artifact handling can be added cleanly in Phase 6.

Blocking severity threshold: high.
</threat_model>

Create `src/capabilities/edit-prompt.ts`.

Implementation requirements:
1. Import OpenAI from `openai`.
2. Import `fs` from `fs`.
3. Export `createEditPromptCapability(): Capability | null`.
4. If `OPENAI_API_KEY` is unset or blank, return `null` so startup still works without OpenAI.
5. Use `process.env.OPENAI_DEFAULT_MODEL?.trim() || 'gpt-image-1'` for the model.
6. Returned capability fields:
   - `op: 'edit_prompt'`
   - `provider: 'openai'`
   - `modelVersion: model`
   - `constraints: { requiresInputImage: true, supportsMultipleInputs: false, maxPromptLength: 4000, supportedSizes: ['square', 'landscape', 'portrait'], outputFormat: 'png' }`
   - `cost: { perCallUsd: 0.04 }`
   - `latencyMsP50: 12000`
   - no `quality` property, leaving `quality.scores` undefined
7. `invoke` must validate:
   - `params.input` is a non-empty string, otherwise throw `new Error("edit_prompt requires params.input file path")`
   - `params.prompt` is a non-empty string, otherwise throw `new Error("edit_prompt requires params.prompt")`
   - prompt length is <= 4000, otherwise throw `new Error("edit_prompt prompt exceeds max length 4000")`
8. Read the input image from disk.
9. Call the OpenAI image edit endpoint with model and prompt. Use the SDK-supported image input shape for the installed OpenAI v4 client. Keep output as base64 JSON.
10. Convert the first returned `b64_json` image to `Buffer`.
11. Return `{ buffer, model, revisedPrompt: imageData.revised_prompt, metadata: { input } }`.
  </action>
