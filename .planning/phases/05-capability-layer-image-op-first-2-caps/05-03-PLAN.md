  <action>
Add `image_op` tool registration inside `registerTools(server)` after `generate_asset`.

Schema:
```ts
op: z.enum(['extract_subject', 'edit_prompt', 'composite_layers', 'transform', 'enhance_upscale', 'analyze_dimensions', 'analyze_palette', 'analyze_ocr', 'generate'])
provider: z.string()
params: z.record(z.unknown()).default({})
outputPath: z.string().optional()
outputDir: z.string().optional()
```

Handler logic:
1. Create `const startedAt = Date.now();`
2. Create a Phase 5 run ID:
   ```ts
   const runId = `run-${new Date().toISOString().replace(/[:.]/g, '-')}-${crypto.randomBytes(3).toString('hex')}`;
   ```
   Add `import * as crypto from 'crypto';` if not already present in `src/index.ts`.
3. Look up capability before validation:
   ```ts
   const capability = capabilityRegistry.get(op as CapabilityOp, provider);
   ```
4. If missing, return JSON text:
   ```json
   {
     "success": false,
     "error": "Capability not registered for op '${op}' and provider '${provider}'",
     "available": capabilityRegistry.list(op as CapabilityOp).map((c) => c.provider)
   }
   ```
   This return must happen before any capability invocation.
5. Call `validateCapabilityParams(capability, params)` before `capability.invoke`.
6. Call:
   ```ts
   const result = await capability.invoke({ params, outputPath, outputDir });
   ```
7. Resolve output path with existing rules:
   ```ts
   const filePath = await resolveOutputPath({
     outputPath,
     outputDir,
     prompt: `${op}-${provider}`,
     provider,
   });
   await saveImage(result.buffer, filePath);
   ```
8. Return JSON text:
   ```json
   {
     "success": true,
     "output": filePath,
     "runId": runId,
     "trace": {
       "nodes": [{
         "id": "n1",
         "op": op,
         "provider": provider,
         "model": result.model,
         "output": filePath,
         "latencyMs": Date.now() - startedAt,
         "revisedPrompt": result.revisedPrompt,
         "metadata": result.metadata
       }]
     }
   }
   ```
9. Catch errors and return:
   ```json
   {
     "success": false,
     "error": message,
     "runId": runId,
     "trace": {
       "nodes": [{
         "id": "n1",
         "op": op,
         "provider": provider,
         "error": message,
         "latencyMs": Date.now() - startedAt
       }]
     }
   }
   ```
10. Do not include base64 image data in any response.
  </action>
