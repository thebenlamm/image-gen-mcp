  <action>
Add the `generate_asset` tool registration in `src/index.ts`. Follow the exact pattern of `generate_image` and `process_image` tools.

1. Add imports at top of file:
   - `import { getPreset, type AssetType } from './utils/presets.js';`
   - `import { applyOperations } from './utils/processing.js';` (already imported — confirm and reuse)

2. Register tool with `server.tool()`:
   - Name: `'generate_asset'`
   - Description: `'Generate a ready-to-use image asset with automatic post-processing. Combines generation and processing in one call.'`
   - Schema (zod):
     - `prompt: z.string()` — "Text description of the image to generate"
     - `assetType: z.enum(['profile_pic', 'post_image', 'hero_photo', 'avatar', 'scene'])` — "Type of asset to generate. profile_pic: 200x200 circular. post_image: 1200x675 16:9. hero_photo: 1080x1920 9:16. avatar: 80x80 circular. scene: 1200x675 16:9."
     - `provider: z.enum(['openai', 'gemini', 'replicate', 'together', 'grok']).optional()` — "Image generation provider"
     - `model: z.string().optional()` — "Provider-specific model"
     - `style: z.string().optional()` — "Style modifier prepended to generation prompt"
     - `assetId: z.string().optional()` — "Asset identifier for clean filename (e.g., 'my-avatar' produces my-avatar.png). Omit for default date/hash naming."
     - `outputPath: z.string().optional()` — "Exact output file path (must end in .png)"
     - `outputDir: z.string().optional()` — "Output directory"

3. Handler implementation:
   a. Look up preset: `const preset = getPreset(assetType as AssetType);`
   b. Resolve provider (same logic as generate_image):
      - Use provider param or DEFAULT_PROVIDER
      - Size-aware selection: if size specified and provider doesn't support, find size-capable
   c. Generate image: `imageProvider.generate({ prompt: effectivePrompt, model, size: preset.generationSize })`
      - Apply style modifier same way: `const effectivePrompt = style ? \`${style}, ${prompt}\` : prompt;`
   d. Apply post-processing: `const processed = await applyOperations(result.buffer, preset.operations);`
   e. Resolve output path: `resolveOutputPath({ outputPath, outputDir, prompt, provider: providerName, assetId })`
   f. Save image: write `processed.buffer` to the resolved path
   g. Return success response (follow generate_image pattern):
      ```json
      {
        "success": true,
        "path": filePath,
        "provider": providerName,
        "model": result.model,
        "assetType": assetType,
        "outputSize": { "width": processed.outputInfo.width, "height": processed.outputInfo.height },
        "operationsApplied": processed.operationsApplied
      }
      ```
   h. Error response (follow generate_image pattern):
      ```json
      {
        "success": false,
        "error": message,
        "provider": providerName,
        "assetType": assetType
      }
      ```

4. Add to startup log: include asset types in console.error output (e.g., `Asset types: profile_pic, post_image, hero_photo, avatar, scene`)

IMPORTANT: The generate_asset tool is placed AFTER the process_image tool registration block.
