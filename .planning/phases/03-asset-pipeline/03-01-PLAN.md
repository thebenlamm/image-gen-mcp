Create `src/utils/presets.ts` with:

2. Define `AssetType` type as union: `'profile_pic' | 'post_image' | 'hero_photo' | 'avatar' | 'scene'`

3. Define `AssetPreset` interface:
   - `assetType: AssetType`
   - `description: string` (human-readable, for tool schema)
   - `generationSize: 'square' | 'landscape' | 'portrait'` (passed to provider)
   - `operations: ProcessingOperation[]` (post-processing pipeline)

   - `profile_pic`: generationSize 'square', operations: [aspectCrop 1:1, circleMask, resize 200x200]

   - `hero_photo`: generationSize 'portrait', operations: [aspectCrop 9:16, resize 1080x1920]
   - `avatar`: generationSize 'square', operations: [aspectCrop 1:1, circleMask, resize 80x80]
   - `scene`: generationSize 'landscape', operations: [aspectCrop 16:9, resize 1200x675]

5. Export `getPreset(assetType: AssetType): AssetPreset` function that returns the preset from `ASSET_PRESETS`.

Note: circleMask goes BEFORE resize for profile_pic and avatar — mask at higher resolution produces smoother edges, then resize down.

   - `outputDir` + `assetId` → `{outputDir}/{assetId}.png`
   - `outputDir` + no assetId → `{outputDir}/{generateFilename(...)}`
   - default dir + `assetId` → `{defaultDir}/{assetId}.png`
   - default dir + no assetId → `{defaultDir}/{generateFilename(...)}`

This is purely additive — when `assetId` is not provided, behavior is identical to current.
