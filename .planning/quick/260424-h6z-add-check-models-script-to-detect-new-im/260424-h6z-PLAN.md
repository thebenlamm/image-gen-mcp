<objective>
Add a `npm run check-models` command that queries each provider's models listing endpoint, filters for image-capable models, and prints a per-provider report comparing discovered models against the defaults currently hard-coded in `src/providers/*.ts`. This makes it trivial to notice when providers ship new image models (e.g. `gpt-image-2`, `gemini-3-pro-image`, `grok-imagine`) so defaults can be bumped.

Purpose: Eliminate the manual "go read OpenAI/Google/xAI/Together changelogs" step. One command, one report.
Output: `scripts/check-models.ts` + wiring in `package.json` + README subsection.
</objective>

  <name>Task 1: Implement scripts/check-models.ts</name>
  <files>scripts/check-models.ts</files>
  <action>
Create `scripts/check-models.ts` as a standalone, read-only Node script (ESM, TypeScript, native fetch only — no new runtime deps).

Structure:

1. `KNOWN_DEFAULTS` constant mirroring current `defaultModel` for each provider (see <interfaces>). Comment clearly that this MUST be kept in sync with `src/providers/*.ts` by hand — the script does not import from src to avoid pulling the MCP SDK at script runtime.

2. Per-provider async fetcher functions, each returning `{ provider: string; skipped?: string; models?: Array<{ id: string; raw?: unknown }> }`:
   - `fetchOpenAI()`: `GET https://api.openai.com/v1/models` with `Authorization: Bearer ${OPENAI_API_KEY}`. Parse `data[].id`.
   - `fetchGemini()`: `GET https://generativelanguage.googleapis.com/v1beta/models?key=${GEMINI_API_KEY}`. Parse `models[].name` (strip `models/` prefix). Include only entries whose `supportedGenerationMethods` includes `generateContent` AND whose name matches the image heuristic, OR whose `outputTokenLimit`/declared modality indicates image — rely on the name heuristic as the primary filter.
   - `fetchGrok()`: Hit BOTH `https://api.x.ai/v1/language-models` and `https://api.x.ai/v1/image-generation-models` with `Authorization: Bearer ${XAI_API_KEY}`. Merge and de-duplicate by id. Image-generation endpoint results are always image-capable; language-models are filtered via the image heuristic (catches aliases like `grok-imagine`).
   - `fetchTogether()`: `GET https://api.together.xyz/v1/models` with `Authorization: Bearer ${TOGETHER_API_KEY}`. Parse the array response (Together returns a bare array). Filter by `type === 'image'` if present, else by the image heuristic on id.
   - `fetchReplicate()`: Return `{ provider: 'replicate', skipped: 'no listing endpoint', models: [] }` unconditionally — Replicate has no simple listing endpoint. Still note the default in the output.

3. For each provider: if the required env var is missing (check `OPENAI_API_KEY`, `GEMINI_API_KEY`, `XAI_API_KEY`, `TOGETHER_API_KEY` — trim like providers do), return `{ provider, skipped: 'no key', models: [] }` without making a network call.

4. Error handling per provider: wrap each fetcher in try/catch; on failure return `{ provider, skipped: \`error: ${err.message}\`, models: [] }`. One provider failing must NOT abort the others. Do NOT swallow silently — the skipped reason must carry the error message (per CLAUDE.md: never swallow exceptions, return errors as observable state).

5. `IMAGE_KEYWORDS` constant: `['image', 'imagen', 'dall-e', 'dalle', 'flux', 'stable-diffusion', 'sd-', 'aurora', 'imagine']`. Helper `isImageModel(id: string): boolean` lowercases id and checks substring. Use only for providers whose listing endpoint does not pre-filter.

6. `main()`: Run all fetchers in parallel via `Promise.all`. Build a report:

   ```
   Image model inventory  —  2026-04-24
   ================================================================

   ## openai                                       default: gpt-image-2
     [default] gpt-image-2
     [ known ] gpt-image-1
     [  NEW  ] gpt-image-3-preview
     [ known ] dall-e-3
     [ known ] dall-e-2

   ## gemini                                       default: gemini-2.5-flash-image
     [default] gemini-2.5-flash-image
     [  NEW  ] gemini-3-pro-image
     ...

   ## grok                                         default: grok-2-image
     [default] grok-2-image
     [  NEW  ] grok-imagine
     ...

   ## together                                     default: black-forest-labs/FLUX.1-schnell
     [default] black-forest-labs/FLUX.1-schnell
     [ known ] black-forest-labs/FLUX.1-dev
     ...

   ## replicate                                    default: black-forest-labs/flux-1.1-pro
     skipped: no listing endpoint  (browse https://replicate.com/collections/text-to-image manually)

   ----------------------------------------------------------------
   Summary: 3 new models detected across 2 providers.
   ```

   Row labels:
   - `[default]` — model id matches `KNOWN_DEFAULTS[provider]`.
   - `[ known ]` — model id was listed in a previous run (treat as `[ known ]` if it's not the default; we don't have persistent state, so any non-default, non-NEW is `known`).
   - `[  NEW  ]` — heuristic: model id contains a version/date substring newer than the default's generation (e.g. id contains `-2`, `-3`, `-4`, `-2026`, `-preview`, `-next`, or `-pro` AND default does not). Keep this conservative: if unsure, mark `[ known ]`. This label is a hint, not a guarantee — the user still eyeballs the report.
   - Actually simpler and less error-prone: mark `[  NEW  ]` for any model id the script can confidently tag as image-capable AND whose generation number (first digit run after provider prefix) is strictly greater than the default's. Fall back to `[ known ]` otherwise. If that heuristic is brittle for a provider, just emit `[ known ]` for everything non-default — the user can spot new names visually. Implement the simpler fallback; a false negative is fine, a false positive on NEW is worse.

7. Exit code: `0` on success (even if some providers skipped). `1` only if ALL providers are skipped or every fetch threw.

8. Shebang + invocation: no shebang needed — invoked via `tsx`. Use top-level `await` inside an `async main(); main().catch(...)` wrapper. Log to stdout via `console.log`; errors/skips via `console.log` too (stdout, not stderr — this is a report, not a log stream).

9. Type safety: define explicit response shapes (`OpenAIModelsResponse`, `GeminiModelsResponse`, `XAIModelsResponse`, `TogetherModel[]`) with the fields we read. Use `as` assertions only at the parse boundary. No `any`.

10. Do NOT import from `src/` — the script must run standalone without compiling the MCP server. Keep `KNOWN_DEFAULTS` local with a comment reminding future maintainers to update it when bumping defaults in providers.

Non-goals (explicitly out of scope for this plan):
- Persisting results to disk or a "last-seen" cache (the NEW heuristic stays stateless).
- Auto-updating `src/providers/*.ts` defaults — this script is read-only per constraints.
- Replicate listing via scraping — intentionally skipped.
